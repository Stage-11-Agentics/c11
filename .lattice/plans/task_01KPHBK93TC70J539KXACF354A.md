# CMUX-7: c11mux summonable guide agent — walks operators through forking and modifying c11mux

Build a c11mux-aware agent an operator (or their agent) can summon from any pane to get walked through the full path from 'I want to change this behavior' to 'my fork is rebuilt and running.' The operator-facing problem: c11mux is meant to be forkable infrastructure for hyperengineers, but the on-ramp from using it to modifying it is undocumented terrain — clone, fork, find the right file, make the change, run ./scripts/reload.sh --tag, verify. Every one of those steps is learnable but not legible. Scope to explore: summoning surface (slash command, cmux guide subcommand, or a skill that is auto-loaded by detection), grounding material (the CLAUDE.md + skill + repo structure + reload flow), and whether the guide is a one-shot walkthrough or a persistent pair-programmer. Voice must match docs/c11mux-voice.md — punchy, lowercase-first, no onboarding warmth. Outcome: forking c11mux feels like a normal operator move, not an advanced one. Connects directly to the spike thesis: the organism builds its own organs.
