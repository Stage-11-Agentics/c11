**Why scope grew (the new finding):**

c11/main does **not** contain `CmuxLayoutNode`, `CmuxPaneDefinition`, `CmuxSplitDefinition`, `CmuxSurfaceDefinition`, or `Workspace.applyCustomLayout()`. They live in upstream `Sources/CmuxConfig.swift` + `Sources/Workspace.swift`. c11 doesn't read `cmux.json` at all — the upstream config-file abstraction isn't present.

So coexistence is not "sniff and route" — it's "translate upstream JSON shape into c11's `WorkspaceApplyPlan`, then route through existing `v2WorkspaceApply`." The translator is real:

- `CmuxLayoutNode.pane(CmuxPaneDefinition{surfaces})` → `LayoutTreeSpec.pane(PaneSpec{surfaceIds})` + collect surfaces into top-level `surfaces[]`
- `CmuxLayoutNode.split({direction, split, children: [a, b]})` → `LayoutTreeSpec.split({orientation, fraction, first, second})` (both schemas validate exactly 2 children)
- `CmuxSurfaceDefinition.{type, name, command, cwd, env, url, focus}` → `SurfaceSpec.{kind, title, command, workingDirectory, env, url, focus-bookkeeping}`
- Generate stable c11 plan-local surface IDs, map env, track focus, validate empty-surface rejection.
